# AgroGuide

AgroGuide is a soil-based fertilizer recommendation system for Tamil Nadu crops. It combines deterministic fertilizer calculation, local document retrieval, and role-based natural-language explanations.

The system supports:

- Rice
- Maize
- Sorghum
- Pearl Millet

It produces different explanations for:

- Farmer
- Agronomist
- Manager

The fertilizer quantities are calculated by code and are not generated by the language model. The language model only explains the final calculated values.

## Features

- Soil-test-based NPK fertilizer recommendation
- Crop-specific constants from `data/constants.json`
- Season-aware target yield for rice
- Local RAG pipeline using TF-IDF retrieval
- Source document chunks displayed for traceability
- Role-based explanation prompts
- pH warnings for acidic or alkaline soil
- Fallback response if the LLM output is incomplete
- Streamlit web interface

## Tech Stack

- Python
- Streamlit
- pypdf
- Scikit-learn TF-IDF retrieval
- SciPy sparse matrix storage
- Groq API for explanation generation

The final RAG implementation uses local TF-IDF retrieval. It does not use Hugging Face embeddings, Chroma, or LangChain.

## File Roles

### `app.py`

Main Streamlit application. It handles:

- UI inputs
- fertilizer calculation call
- retrieval query creation
- top-4 document chunk retrieval
- role-based prompt construction
- Groq API call
- fallback response
- displaying metrics, recommendation, warnings, and source chunks

### `calculator.py`

Deterministic fertilizer calculation engine. It:

- loads `data/constants.json`
- computes crop nutrient requirements
- subtracts available soil nutrients
- applies nutrient use efficiency
- converts nutrient requirements to fertilizer material quantities
- scales recommendations by field area
- returns warnings for pH and edge cases

### `ingest.py`

Document ingestion script. It:

- reads all PDF files from `documents/`
- extracts text page by page using `pypdf`
- cleans extracted text
- chunks text into 500-word chunks with 50-word overlap
- builds a TF-IDF index using Scikit-learn
- saves retrieval files to `vectorstore/`

Run this script whenever documents are added or changed.

### `prompts.py`

Role-based prompt templates for:

- Farmer
- Agronomist
- Manager

Each template includes a guardrail telling the model not to change the calculated fertilizer quantities.

### `data/constants.json`

Stores crop constants and fertilizer parameters, including:

- crop target yield
- rice target yield by season
- nutrient required per ton
- nutrient use efficiency
- fertilizer nutrient fractions

## Fertilizer Formula

For each nutrient:

```text
Total nutrient requirement = Target yield x Nutrient per ton
```

```text
Net nutrient requirement = (Total requirement - Soil available nutrient) / Efficiency
```

Negative nutrient requirements are clipped to zero.

Fertilizer conversion:

```text
Urea = Net N / 0.46
SSP  = Net P / 0.16
MOP  = Net K / 0.60
```

Area conversion:

```text
1 acre = 0.404686 hectare
Final fertilizer quantity = per-hectare dose x field area in hectares
```

## Setup

Open PowerShell in the project folder:

```powershell
cd AgroGuide
```

Create and activate a virtual environment if needed:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

If PowerShell blocks activation:

```powershell

.\.venv\Scripts\Activate.ps1
```

Install dependencies:

```powershell
python -m pip install --upgrade pip setuptools wheel
python -m pip install -r requirements.txt
```

## Environment Variables

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
```

Do not commit or share your real API key.

## Build the Retrieval Index

Run ingestion once before starting the app:

```powershell
python ingest.py
```

This creates the local retrieval index inside `vectorstore/`.

Run ingestion again whenever you add, remove, or update PDFs in `documents/`.

## How RAG Works

1. PDFs are processed by `ingest.py`.
2. Text is split into chunks.
3. Chunks are vectorized using TF-IDF.
4. At runtime, the app builds a query using crop, district, season, and soil type.
5. The top 4 matching chunks are retrieved.
6. Retrieved chunks are included in the role-specific prompt.
7. The response and source chunks are displayed to the user.

## Supported Inputs

Farm context:

- Crop
- District
- Soil type
- Season
- Role

Soil and field inputs:

- Soil N in kg/ha
- Soil P in kg/ha
- Soil K in kg/ha
- Soil pH
- Organic carbon percentage
- Field area in acres

## Notes

- Soil N, P, and K directly affect fertilizer quantities.
- Soil pH is used for warnings.
- Organic carbon is retained as explanation context.
- If all soil N, P, and K values are zero, the calculator uses default average fertility values.
- If a nutrient is already sufficient in the soil, the corresponding fertilizer dose becomes `0.0 kg`.

## Troubleshooting

### Vector store not found

Run:

```powershell
python ingest.py
```

### Missing Groq API key

Check that `.env` contains:

```env
GROQ_API_KEY=your_groq_api_key_here
```

Then restart Streamlit.

### Package issues

Use a clean virtual environment:

```powershell
Remove-Item -Recurse -Force .venv
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip setuptools wheel
python -m pip install -r requirements.txt
```

### App still shows old behavior

Stop Streamlit with `Ctrl + C`, then restart:

```powershell
streamlit run app.py
```

## Important Design Point

AgroGuide is not a prompt-only chatbot. It first calculates fertilizer quantities using deterministic formulas, then retrieves document context, and only then asks the language model to explain the fixed recommendation.

This makes the system more traceable and safer than asking an LLM to invent fertilizer quantities directly.
